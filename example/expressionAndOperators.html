Operator Association

It specifies the order in which the operators of the same precedence are performed. Left-to-right association
means that operations are performed from left to right. Example:
left-to-right

  • w = x - y - z; ==== is same as ==== ((x - y) - z);

right-to-left

  • x = ~-y; ==== is same as ==== x = ~(-y);
  • w = x = y = z; ===== is same as ==== w = (x = (y = z));
  • q = a?b:c?d:e?f:g; ==== is same as ==== q = (a?b:(c?d:(e?f:g)));
Order of evaluation only makes a difference if any of the subexpressions have a side effect that affects other expression!

+ operator

The conversion rules for + gives priority to string concatenation:

  1. If ether operands are objects: it converts to primitives using either toString() or valueOf()
  2. After object-to-primitive if either operand is string the other is converted to String and string concatenation is performed.
  3. Otherwise both operands are converted to numbers or NaN and addition is performed.
